          v0.7.9 — current
          Local proxy for Greysheet CORS
        

        

          
+coinfalcon_proxy.py — companion Python proxy server (85 lines, Flask). Forwards browser requests to Greysheet API with CORS headers, resolving the browser CORS block. Supports both Greysheet and PCGS targets. Run with: pip install flask requests && python3 coinfalcon_proxy.py. Listens on http://localhost:3001 by default; optional port arg: python3 coinfalcon_proxy.py 8080.

          
+Proxy URL field — new Proxy URL input in the Greysheet sidebar section. Enter http://localhost:3001 when the proxy is running. The greysheetUrl() helper routes all Greysheet API calls through the proxy when configured, or attempts direct calls (CORS-blocked) when empty. Test Greysheet button now shows a clear warning if proxy URL is not configured.

        

      

      

      

        

          v0.7.8
          API status panel + connection tests
        

        

          
+API Status panel — new section at top of sidebar shows four live status dots: Anthropic · Greysheet · PCGS · Numista. Dots update on test runs: grey (not configured), amber pulsing (testing), green (connected), red (error). A ⚙ Test all connections button runs all four tests in parallel.

          
+Per-API test buttons — each API section now has a ⚙ Test connection button. Greysheet: calls GetNodeRequest?NodeId=1 and reports the node name on success, or specific error (auth failed / CORS blocked / subscription required). PCGS: calls GetCoinFactsByPCGSNoAndGrade for PCGS#7130 MS-63 (1881-S Morgan) and shows the returned price. Numista: searches for Morgan Dollar and reports result count.

          
~CORS detection — if an API call fails with a network/fetch error (typical of browser CORS rejection), the status shows a specific "CORS blocked — may require a backend proxy" message rather than a generic error.

        

      

      

      

        

          v0.7.7
          CDN Greysheet API integration
        

        

          
+Greysheet API — Tier 0A (best source) — CDN Greysheet API integrated as the highest-priority pricing source. Returns five price points in a single call: Greysheet wholesale bid (GreyVal1), CPG retail (CpgVal), PCGS Price Guide (PcgsVal1), NGC Price Guide (NgcVal1), Blue Book (BlueBookVal1), plus CAC sticker flag. The wholesale bid is the industry standard dealer-to-dealer buy price used by Heritage, Stack’s Bowers, and David Lawrence. Requires Coin Dealer Digital subscription and API key (base64 encoded username:password) entered in new sidebar section.

          
+Dealer spread display — when Greysheet data is available, the value card shows the wholesale bid as the main figure, the CPG retail above it, and the dollar spread between them (“+$40 dealer spread → $185 CPG retail”). This is the core buying-decision intelligence for a dealer: what to pay vs what to resell for.

          
~Pricing tier hierarchy updated — Tier 0A: Greysheet API (new) · Tier 0B: PCGS API (demoted) · Tier 1: web search batch · Tier 2: coinfoxa.com. CSV adds greysheet_bid, greysheet_cpg_retail, greysheet_pcgs_guide, greysheet_ngc_guide, greysheet_blue_book, greysheet_cac columns.

        

      

      

      

        

          v0.7.6
          eBay date gate removed from prompt
        

        

          
✕Root cause of "No qualifying eBay sale" bug — the Tier 1 prompt included a 90-day date filter that the model was applying even when web search results did not show explicit sale dates. When the model could not confirm a date it reported "No qualifying eBay completed sale within 90-day window found" and withheld the eBay price, defeating the purpose.

          
+Correct separation of concerns — the model now returns whatever eBay data it finds with the sale date in found_date (YYYY-MM-DD) if determinable, null otherwise. The client-side isStalePriceData() function performs the 90-day check and shows the ⚠ STALE badge when appropriate. The model finds prices; the app assesses recency.

        

      

      

      

        

          v0.7.5
          Pricing prompt regression fix
        

        

          
✕Regression in v0.7.4 fixed — the 90-day eBay date rule was written as a gate that caused the model to return null prices for all USD fields when it could not find recent eBay data, even though PCGS/NGC price guide data was available. Prompts rewritten so the date restriction is advisory for eBay only: PCGS and NGC prices are always returned regardless of eBay availability. Coinfoxa Tier 2 now returns the best available price regardless of age and relies on the client-side isStalePriceData() check to flag it.

        

      

      

      

        

          v0.7.4
          Date-constrained pricing + stale data detection
        

        

          
+90-day eBay date constraint — Tier 1 and Tier 2 prompts now explicitly compute a cutoff date (today minus 90 days) and instruct the model to reject any eBay or auction sale older than that date. PCGS and NGC price guides are exempt (they are always current). If the most recent eBay data found is older than the cutoff the model is instructed to report it as unavailable rather than using stale figures.

          
+found_date field in price response — the model now returns the actual date of the eBay sale it found (YYYY-MM-DD) as a found_date field. Shown in the value card source line: "◆ eBay recent sales · data from 2026-04-15".

          
+⚠ STALE badge + warning — if found_date is over 90 days old (client-side check, not model-dependent), the value card shows an amber "⚠ STALE" badge and the warning "eBay data from [date] is over 90 days old — verify current pricing". PCGS Price Guide results are never flagged stale. Stale flag exported to CSV as live_value_stale column.

        

      

      

      

        

          v0.7.3
          Numista catalog refs fix
        

        

          
+Numista catalog reference display fixed — Numista API v3 returns the catalogue field as an object {"id":"km","name":"KM#"} rather than a plain string. The map callback was serializing this as [object Object]. Fixed to use ref.catalogue?.name || ref.catalogue?.id, correctly displaying KM 108 etc.

        

      

      

      

        

          v0.7.2
          PDF text encoding + overflow fix
        

        

          
+pdfSafe() text sanitizer — new helper strips and replaces characters outside jsPDF WinAnsi encoding before any text reaches the PDF renderer. Fixes %Æ garbage from em-dashes, bullets, smart quotes, arrows, and Unicode symbols the AI commonly outputs in recommendation reasons and observations. Converts to ASCII equivalents: — becomes -, • becomes *, … becomes ..., etc.

          
+PDF lower section overflow fix — recommendation reason reduced to 2 lines at 7.5pt (was 3 lines at 8.5pt), key observations reduced to 3 items at 7.5pt with tighter line spacing. Both sections have stricter page-bottom guards (PH-40 and PH-38 respectively) preventing text from overlapping the footer.

        

      

      

      

        

          v0.7.1
          PCGS API pricing + batch price fix
        

        

          
+PCGS API Tier 0 pricing — new PCGS Access Token field in sidebar. For US coins where the AI identified a PCGS number, the PCGS Public API (GetCoinFactsByPCGSNoAndGrade) is called directly, returning the authoritative official retail price. This fires in parallel before Tier 1 web search. Coins priced via PCGS API skip web search entirely. Coins without a PCGS number fall through to Tier 1 as before.

          
+Batch price lookup fixed — matchByIndex now has a positional fallback: if the model returns results in order without coin_index fields, they are matched by array position. This fixes the "No pricing found" error on common coins where the batch web search was finding prices but the index matching was failing silently.

          
~Batch prompt simplified — Tier 1 prompt no longer asks for coin_index in the JSON response (which was causing matching failures). Results are expected in the same order as the input list. max_tokens raised 1024→1500. Added console logging of matched prices for debugging.

        

      

      

      

        

          v0.7.0
          Slab detection + certification number
        

        

          
+Slab detection — grading prompt now identifies whether each coin is raw or in a professional holder (PCGS, NGC, ANACS, ICG, other). For slabbed coins the label grade is used directly — the model does not re-assess. Holder type shown in flag pills, panel header service badge, and the new Holder Status card in the details panel.

+Special designations — CAC sticker, + grade, PL, DMPL, Cameo, Deep Cameo, Star detected from label and shown as gold flag badges.

          
+Price warning for slabbed coins — certification banner includes "⚠ Price shown is raw estimate" note, since the certified market price (via PCGS/NGC cert number API) is a Phase 2 feature.

          
~CSV + PDF updated — holder_type, holder_service, cert_number, label_grade, label_designation added to CSV. PDF coin pages show certification banner with cert number, and the summary table includes cert service + number in the coin name column.

        

      

      

      

        

          v0.6.9
          Numista catalog integration
        

        

          
+Scenario 1 — Catalog identification — after grading, a Numista lookup fires automatically for every coin (requires Numista API key in sidebar). Searches by series + country + year, fetches the matched type detail. Returns KM and other catalog reference numbers, year range, mintage, edge type, orientation, and diameter. Particularly strong for world and foreign coins where catalog identification adds significant dealer value.

          
+Scenario 2 — Metal composition verification — Numista's catalog fineness and weight are cross-checked against the AI's precious_metal estimates. If discrepancy exceeds 2%, a note is added ("Numista verified: 26.73g × 90.0% = 0.77344 oz · AI est was 0.77200 oz"). Confirmation or correction exported to CSV as numista_melt_note.

          
+Scenario 3 — Reference images — Numista obverse and reverse reference images displayed in the Numista card. 66px thumbnails with graceful fallback if images are unavailable. Direct link to the coin's Numista catalog page.

          
~API details — numista.com free tier, 2,000 requests/month. 2 calls per coin (search + detail). Auth via Numista-API-Key header. Three-pass search strategy: series+country+year → series+country → series only. Key entered in new Numista API sidebar section. 300ms stagger between concurrent lookups.

        

      

      

      

        

          v0.6.8
          Image quality pre-flight checker
        

        

          
+4-check image quality pre-flight — runs automatically after image upload, purely client-side (Canvas API, no API calls, no cost). Checks: (1) Resolution — shortest side ≥800px required for date digit reads; (2) Aspect ratio — warns if >1.5:1, errors >2.2:1; (3) Overexposure — samples full histogram, warns >10% blown pixels, errors >25%; (4) Edge contrast — Sobel-like center-region gradient analysis detects flat overhead lighting vs angled raking light.

          
+Pre-flight panel UI — appears between image preview and Analyze button. Per-image breakdown with ✓ / ⚠ / ✕ per check, pixel dimensions, and specific remediation guidance. Dismissible. Analyze button always remains clickable — quality checks inform, never block.

          
~Analyze button reflects quality — button label and border change to amber with issue count when errors are found, restoring to normal after Clear or new upload.

+Full session history in sidebar — history now accumulates all coins from every analysis run this session, not just the most recent batch. Each analysis is shown under a labeled divider (Analysis 1 · 3 coins, Analysis 2 · 2 coins, etc.) so batches are clearly separated. Value column uses live price when available, falls back to AI estimate. Recommendation color-coded green/amber/red.

          
~Scrollable history panel — history list has max-height 320px with scroll so it does not push other sidebar sections off screen. Auto-scrolls to latest coins after each analysis. Clears correctly with the Clear button.

        

      

      

      

        

          v0.6.6
          Overloaded error handling
        

        

          
+Extended overloaded retry — 4 attempts — overloaded_error now retries up to 4 times at 15s · 30s · 60s · 90s intervals (total max wait ~3.3 min). Rate limit errors unchanged at 2 retries. Status bar shows "retry N/4 in Xs…" countdown.

          
+↺ Try Again button — if all retries fail, an amber Try Again button appears in the status bar. Re-runs analysis with the same images already loaded — no need to re-upload. Button hides automatically when analysis starts.

          
+Human-readable error message — overloaded errors now read "Anthropic API is temporarily at capacity (server-side issue, not your connection)" instead of the raw API error string.

        

      

      

      

        

          v0.6.5
          Manual price lookup + overloaded error retry
        

        

          
+Fetch Live Prices is now manual — price lookups no longer fire automatically after grading. Grading result renders immediately with AI estimates. A ⟳ Fetch Live Prices button in the export row triggers the two-tier lookup (PCGS·NGC·eBay then coinfoxa.com) only when pressed. This eliminates unintended API usage and the overloaded errors caused by auto-firing multiple calls immediately after a grading call.

          
+Overloaded error retry — both fireGradingCall() and runPriceSearch() now retry on overloaded_error (Anthropic API temporarily at capacity) in addition to rate_limit_error. Overloaded retries wait 20s / 40s (longer than rate limit 12s / 24s) since capacity recovery takes more time.

          
~Value card default state — cards now show the AI estimate cleanly on load rather than a loading spinner. The "AI estimate · press Fetch Live Prices" hint makes the data source explicit before any live lookup is triggered.

          v0.6.3
          Adaptive image resolution toggle
        

        

          
+High-res images toggle — new sidebar toggle under Output. Off (default): 768px — fast, rate-limit safe, appropriate for bulk circulated coin valuation. On: 1024px — better hairline detection, more reliable MS-66+ distinctions, improved accuracy on small coins and faint dates. Input tokens ~44% higher in high-res mode.

          
~Status bar reflects mode — compression status shows "768px" or "1024px (high-res)" so it is always clear which mode was used for a given analysis.

        

      

      

      

        

          v0.6.2
          Rate limit hardening
        

        

          
+Retry on grading call — fireGradingCall() wraps the main vision API call with the same 12s/24s exponential backoff used by price lookups. Rate limit errors on the grading call now auto-retry instead of surfacing as hard errors.

          
+10-second price lookup delay — price batch calls fire 10 seconds after grading completes, ensuring the grading call's token usage has cleared the rolling 60-second rate limit window before the next API call fires.

          
+Image resize 1024→768px — coin images compressed to 768px maximum before encoding. Reduces image input tokens by ~44% per image (768² vs 1024² pixels). Grading accuracy is unaffected — 768px is sufficient resolution for coin surface detail at Claude's vision capability level.

          
iLong-term: contact Anthropic sales to upgrade the org rate limit. At production scale ($149/month SaaS, 500 coins/month) a higher tier limit is the correct solution. Current limit: 30k input tokens/minute.

        

      

      

      

        

          v0.6.1
          Batched pricing — rate limit fix
        

        

          
+Batched price lookups — value lookups now use a maximum of 2 API calls regardless of how many coins are in the image. Tier 1 sends all coins in a single batch request (PCGS · NGC · eBay); Tier 2 sends only the coins that missed Tier 1 in a single coinfoxa.com batch request. Previously: up to 12 calls for a 6-coin image. Now: always 2. Eliminates the 30,000 input token/minute rate limit error.

          
+Automatic retry on rate limit — if a 429 rate_limit_error is returned, the call retries automatically after 12s (first retry) or 24s (second retry) with a status bar countdown. After 2 retries, shows a clear error message.

          
~Architecture — shared valueLookupHeaders(), runPriceSearch(prompt, maxTokens, attempt), priceIsValid(), buildCoinDesc(), matchByIndex() helpers. fetchAllMarketValues() now orchestrates both tiers sequentially with clean index matching.

        

      

      

      

        

          v0.6.0
          coinfoxa.com two-tier pricing
        

        

          
+coinfoxa.com as Tier 2 price source — when the primary lookup (PCGS · NGC · eBay) returns no price, a second web search call fires automatically targeting coinfoxa.com via site:coinfoxa.com. Coinfoxa aggregates worldwide auction results and is particularly effective for foreign and world coins not covered by US-centric price guides. Fires only on primary miss — no extra cost for coins where primary succeeds.

          
+Distinct coinfoxa visual — value cards sourced from coinfoxa.com show a blue ● LIVE badge and blue border (vs green for PCGS/NGC/eBay) so the source is immediately identifiable at a glance.

          
+Intermediate search state — when Tier 1 misses, the value card updates to show "Searching coinfoxa.com…" before the Tier 2 call fires, giving visual feedback that lookup is continuing rather than failed.

          
~Refactored lookup architecture — shared valueLookupHeaders() and runPriceSearch() helpers eliminate duplicated code between tiers. updateValueCard() accepts isFoxa flag for source-aware styling.

v0.5.0CSV export + grid position mapping
        

        

          
+CSV export — ⬇ Export CSV button appears after every analysis. Generates a flat .csv with one row per coin and 57 columns covering all grading data, melt value formula, spot prices at time of analysis, image filenames, and timestamp. Compatible with Excel, Google Sheets, and any dealer inventory system.

          
+Grid position mapping — Each coin is assigned an Excel-style grid coordinate (A1, B2, C3...) based on its physical position in the image. Column = letter left-to-right, row = number top-to-bottom. A1 is always top-left. Shown in panel headers, summary pills, and exported to CSV. Enables physical tray mapping for bulk dealer lots.

          
~Prompt updated — Step 1 (Coin Detection) now includes grid position assignment instructions. JSON schema adds grid_position as first field per coin object.

          
~API fixes — Added anthropic-version and anthropic-dangerous-direct-browser-access headers for local file:// execution. Upgraded model to claude-sonnet-4-6. max_tokens raised 4096→8192. Diagnostic button added to sidebar for API connection testing.
